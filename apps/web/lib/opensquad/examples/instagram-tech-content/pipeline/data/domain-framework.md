# Domain Framework — Produção de Carrosséis Instagram para Tech/IA

## Metodologia Operacional

### Fase 1: Pesquisa e Curadoria de Notícias

1. Mapear fontes de notícias tech/IA (TechCrunch, The Verge, Ars Technica, blogs de IA, releases oficiais)
2. Buscar notícias recentes dentro do período definido pelo usuário
3. Filtrar por relevância para a comunidade tech brasileira
4. Rankear por potencial de engajamento: novidade × impacto × controversia
5. Apresentar top 3-5 histórias com título, fonte, data e resumo

### Fase 2: Geração de Ângulos

1. A partir da notícia selecionada, aplicar os 5 ângulos de copywriting:
   - 🔴 Medo: consequência negativa de ignorar
   - 🟢 Oportunidade: janela de vantagem competitiva
   - 📚 Educacional: explicação prática com demonstração
   - ↔️ Contrário: perspectiva que desafia o consenso
   - ⭐ Inspiracional: visão de futuro que motiva
2. Para cada ângulo, criar hook + premissa em 1-2 frases
3. Apresentar todos os ângulos para escolha do usuário

### Fase 3: Criação do Carrossel

1. Selecionar formato de carrossel apropriado (Editorial, Listicle, Tutorial, Mito vs Realidade, Storytelling, Problema → Solução, Antes e Depois)
2. Aplicar diagnóstico pré-escrita:
   - Nível de consciência do público (Schwartz)
   - Nível de sofisticação do mercado
   - Big Idea (inimigo, mecanismo, promessa)
   - Driver psicológico dominante
3. Desenvolver 8-10 slides com hierarquia de 2 camadas (headline bold + texto de suporte)
4. Garantir 40-80 palavras por slide
5. Alternar cores de fundo (claro, escuro, destaque)
6. Criar caption com hook nos primeiros 125 caracteres
7. Fechar com CTA específico e hashtags (5-15, mix de nicho e broad)

### Fase 4: Otimização

1. Aplicar Copy Stress Test (crédibilidade, provas, inflação, fricção)
2. Reduzir word count em 15-25% sem perder substância
3. Verificar scroll-stop test no hook
4. Anti-commodity check: conteúdo não pode ser usado por concorrente
5. Validar constraints técnicos (2200 chars caption, 125 chars visíveis, ratio 3:4)

### Fase 5: Review e Aprovação

1. Avaliar contra critérios de qualidade definidos (scoring 1-10 por critério)
2. Verificar alinhamento com tom de voz da marca
3. Validar formatação e constraints da plataforma
4. Emitir veredicto: APPROVE (≥7/10) / REJECT (<7/10 ou qualquer critério <4/10)
5. Se REJECT: fornecer fixes específicos e acionáveis

## Critérios de Decisão

- **Quando usar formato Editorial**: tendências, análises de mercado, takes contrários, argumentos com dados
- **Quando usar formato Listicle**: dicas, ferramentas, recursos, recomendações
- **Quando usar formato Tutorial**: guias passo-a-passo, workflows, configurações
- **Quando usar formato Storytelling**: histórias pessoais, jornadas de cliente, lições aprendidas
- **Quando usar formato Problema → Solução**: posicionamento de produto, frustrações comuns
- **Quando escolher 8 slides vs 10+**: 8 para temas diretos, 10+ para deep-dives educacionais
- **Quando hookear com pergunta vs statement**: pergunta para audiência morna, statement bold para fria
